# Test Requirements: Server-Side Lineup Carry-Forward

## Automated Tests

### Unit Tests (`tests/unit/ensure-lineups.test.ts`)

| AC | Criterion | Test Description |
|----|-----------|-----------------|
| AC1.1 | Carry-forward clones previous GW lineup | Mock team with GW1 lineup (11 XI + 4 bench + captain/VC). Call ensureLineups for GW2. Verify created lineup has identical playerId, slotType, benchPriority, role for all 15 slots. |
| AC1.2 | Picks most recent previous lineup | Mock team with GW1 and GW2 lineups (different captains). Call ensureLineups for GW3. Verify carried-forward lineup matches GW2 (not GW1). |
| AC1.4 | Skips team with existing lineup | Mock team with lineup already set for current GW. Call ensureLineups. Verify no new lineup created, lineups array unchanged, carriedForward = 0. |
| AC1.5 | Removed player excluded | Mock team with GW1 lineup containing player X. Player X not in current TeamPlayer squad. Call ensureLineups for GW2. Verify carried-forward lineup excludes player X's slot. |
| AC2.1 | Auto-generate from squad | Mock team with 15 TeamPlayers (varying purchasePrice), no lineups. Call ensureLineups. Verify lineup created with 15 slots. |
| AC2.2 | XI/Bench by purchasePrice | Mock 15 players with prices 15.0 down to 1.0. Verify top 11 have slotType 'XI', bottom 4 have slotType 'BENCH'. |
| AC2.3 | Captain = highest price | Verify player with price 15.0 has role 'CAPTAIN', player with price 14.0 has role 'VC'. |
| AC2.4 | Bench priority order | Verify bench player with price 4.0 has benchPriority 1, price 3.0 = priority 2, etc. |
| AC2.5 | Skip team with no squad | Mock team with 0 TeamPlayers. Call ensureLineups. Verify no lineup created, autoGenerated = 0. |
| AC2.6 | Carry-forward takes precedence | Mock team with previous lineup AND squad. Call ensureLineups. Verify carried forward (not auto-generated) — slots match previous lineup, not purchasePrice order. |
