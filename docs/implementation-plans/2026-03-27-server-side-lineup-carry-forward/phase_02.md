**Codebase verified:** 2026-03-27

## Acceptance Criteria Coverage

This phase implements:

### server-side-lineup-carry-forward.AC2: Auto-Generate from Squad
- **server-side-lineup-carry-forward.AC2.1 Success:** Team with no lineup in any gameweek but with a 15-player squad gets an auto-generated lineup
- **server-side-lineup-carry-forward.AC2.2 Success:** Top 11 players by `purchasePrice` are placed in XI (`slotType: 'XI'`), remaining 4 on bench (`slotType: 'BENCH'`)
- **server-side-lineup-carry-forward.AC2.3 Success:** Captain = highest `purchasePrice` (`role: 'CAPTAIN'`), Vice Captain = second highest (`role: 'VC'`)
- **server-side-lineup-carry-forward.AC2.4 Success:** Bench priority assigned by descending `purchasePrice` (priority 1 = most expensive bench player, priority 4 = least)
- **server-side-lineup-carry-forward.AC2.5 Failure:** Team with no squad (0 TeamPlayers) is skipped — no lineup created
- **server-side-lineup-carry-forward.AC2.6 Failure:** Team that already received a carry-forward lineup in Phase 1 is not auto-generated

### Task 1: Add auto-generate logic to `ensureLineups()`

**Verifies:** server-side-lineup-carry-forward.AC2.1, server-side-lineup-carry-forward.AC2.2, server-side-lineup-carry-forward.AC2.3, server-side-lineup-carry-forward.AC2.4, server-side-lineup-carry-forward.AC2.5, server-side-lineup-carry-forward.AC2.6

**Files:**
- Modify: `lib/scoring/pipeline.ts` — extend `ensureLineups()` function to handle Case 2

**Implementation:**

After the carry-forward logic (Case 1) in `ensureLineups()`, add Case 2 for teams that STILL have no lineup after carry-forward attempted (no previous lineup exists anywhere):

1. For each team where `team.lineups[0]` is still undefined after the carry-forward pass:
2. Query `tx.teamPlayer.findMany({ where: { teamId: team.id }, include: { player: true }, orderBy: { purchasePrice: 'desc' } })`
3. If squad is empty (0 players), skip this team (AC2.5)
4. Take top 11 by `purchasePrice` → create `LineupSlot` with `slotType: 'XI'` (AC2.2)
5. Remaining players (up to 4) → create `LineupSlot` with `slotType: 'BENCH'`, `benchPriority` 1, 2, 3, 4 ordered by descending price (AC2.4)
6. First player (index 0, highest price) gets `role: 'CAPTAIN'` (AC2.3)
7. Second player (index 1, second highest price) gets `role: 'VC'` (AC2.3)
8. All other players get `role: null`
9. Create Lineup + LineupSlot records via `tx.lineup.create({ data: { teamId, gameweekId, slots: { create: [...] } } })`
10. Push into `team.lineups` array so scoring loop picks it up
11. Log: `console.log(\`ensureLineups: auto-generated ${team.name} (captain: ${captain.player.fullname})\`)`
12. Increment `autoGenerated` counter

Key details:
- Captain and VC are always in XI (they're the top 2 by price, always in the top 11)
- `purchasePrice` is `Float @default(0)` — sort descending, ties broken by Prisma default ordering
- If fewer than 15 players: put as many as possible in XI (up to 11), rest on bench
- Teams that got a carry-forward in the same `ensureLineups()` call already have `team.lineups[0]` populated, so they're skipped here (AC2.6)

**Testing:**
Tests must verify:
- server-side-lineup-carry-forward.AC2.1: Team with squad but no lineups gets auto-generated lineup with 15 slots
- server-side-lineup-carry-forward.AC2.2: Top 11 by price have `slotType: 'XI'`, bottom 4 have `slotType: 'BENCH'`
- server-side-lineup-carry-forward.AC2.3: Highest price player has `role: 'CAPTAIN'`, second has `role: 'VC'`
- server-side-lineup-carry-forward.AC2.4: Bench priority 1 = most expensive bench player, 4 = least
- server-side-lineup-carry-forward.AC2.5: Team with 0 TeamPlayers → no lineup created, autoGenerated stays 0
- server-side-lineup-carry-forward.AC2.6: Team that got a carry-forward in same run is not auto-generated
