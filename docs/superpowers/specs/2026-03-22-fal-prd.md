# FAL — Fantasy Auction League: Product Requirements Document

**Version:** 1.0
**Date:** 22 March 2026
**Status:** Phase 1 Active Development

---

## 1. Executive Summary

FAL (Fantasy Auction League) is a private, invite-only fantasy cricket platform for IPL built for friends who want more depth and strategy than mainstream apps like Dream11. Teams are assembled via a pre-season auction, squads are locked for the season, and managers make weekly lineup decisions to maximise points from real IPL match performance.

Unlike per-gameweek platforms, FAL's auction model creates genuine season-long investment in players, meaningful chip strategy decisions, and a richer social dynamic between a fixed group of managers.

**Phase 1 goal:** Deliver a fully playable season for 2–15 managers — auction management, weekly lineups, automated scoring, and a live leaderboard — on zero infrastructure cost (Vercel Hobby + Neon free tier).

## 3. Users & Roles

### 3.1 League Admin
- Creates the league and generates an invite code
- Uploads team rosters (CSV) after the pre-season auction
- Triggers score imports after each IPL match (on-demand button)
- Manages manager list (can remove managers)
- Also participates as a team manager

### 3.2 Team Manager
- Joins via invite code
- Sets weekly lineup: Playing XI, Captain, Vice Captain, bench order
- Activates one strategy chip per gameweek (once per season each)
- Views leaderboard, standings, and other managers' lineups

### 3.3 Constraints
| Rule | Detail |
|---|---|
| Max managers per league | 15 |
| Min managers per league | 2 |
| Squad size | 12–15 players |
| Player uniqueness | A player can only be on one team within a league |
| Season start gate | Admin cannot start the season until all teams have a valid squad (min 12 players) |

## 4. Core Game Loop

```
Pre-Season Auction
       ↓
Admin uploads rosters via CSV
       ↓
┌──────────────────────────────────────────────┐
│              GAMEWEEK CYCLE (×10)            │
│                                              │
│  Deadline opens → Managers submit lineup     │
│       + Captain / VC / optional chip         │
│              ↓                               │
│  Lock: First IPL match of the GW starts      │
│              ↓                               │
│  IPL matches played (Mon–Sun)                │
│              ↓                               │
│  Admin triggers score import after each match│
│              ↓                               │
│  Bench subs → Multipliers → Chips → Totals   │
│              ↓                               │
│  Leaderboard updated                         │
└──────────────────────────────────────────────┘
       ↓
Season ends → Final standings
```

### Lineup Lock
- Trigger: exact start time of the first IPL match in that gameweek
- After lock: no edits to XI, Captain, VC, bench order, or chip selection
- No submission: previous gameweek's lineup carries forward automatically (first week with no previous = 0 points)

## 5. Team Composition & Lineup Rules

### 5.1 Squad Structure
| Component | Size |
|---|---|
| Full squad | 12–15 players |
| Playing XI | 11 (selected from squad each week) |
| Bench | 1–4 remaining squad players |

### 5.2 Weekly Submission
Each gameweek, managers submit:
- **Playing XI** — 11 players from their squad
- **Captain** — earns 2× points for the gameweek
- **Vice Captain** — earns 2× points **only if the Captain did not play** (not in the Playing XI and not as an Impact Sub) across the entire gameweek. Otherwise earns standard 1× points like any other player.
- **Bench Priority Order** — Bench 1 through 4, used for auto-substitution
- **Strategy Chip** (optional) — one of 2 chips, each usable once per season

### 5.3 Bench Auto-Substitution
Triggered when a Playing XI player does not appear in any match during the gameweek.

**Resolution logic:**
1. Check bench in priority order (1 → 2 → 3 → 4)
2. First bench player who played in at least one match is substituted in
3. If no bench player played → that slot scores 0

**Edge cases:**
| Scenario | Handling |
|---|---|
| Captain didn't play | Bench sub fills the XI slot. VC is promoted to 2× — this is the only scenario where the VC earns a multiplier. |
| VC didn't play | Bench sub fills XI slot. VC earns no points (and no multiplier — VC multiplier only triggers if Captain is absent). |
| Both Captain and VC didn't play | No multipliers applied to anyone. |
| Multiple XI players absent | Each gets a separate bench sub in priority order. No double-dipping on a single bench player. |

## 6. Scoring System

Aligned with Dream11 T20 scoring as the industry baseline.

### 6.1 Batting

| Event | Points | Notes |
|---|---|---|
| Run scored | +1 | |
| Boundary bonus (four) | +4 | |
| Six bonus | +6 | |
| 25-run milestone | +4 | |
| Half-century (50) | +8 | |
| 75-run milestone | +12 | |
| Century (100) | +16 | Replaces all lower milestones — no stacking |
| Duck | −2 | BAT, WK, ALL only — bowlers exempt. Must have faced ≥1 ball. |

**Milestone stacking rule:** Below 100, milestones stack (75 runs = +4 + +8 + +12 = +24 bonus). A century earns +16 only — the 25/50/75 bonuses are not also awarded.

**Strike Rate bonus/penalty** (min 10 balls faced; bowlers exempt):
| Strike Rate | Points |
|---|---|
| Above 170 | +6 |
| 150.01 – 170 | +4 |
| 130 – 150 | +2 |
| 70.01 – 129.99 | 0 |
| 60 – 70 | −2 |
| 50 – 59.99 | −4 |
| Below 50 | −6 |

### 6.2 Bowling

| Event | Points | Notes |
|---|---|---|
| Dot ball | +1 | Requires ball-by-ball data |
| Wicket (excl. run out) | +30 | |
| LBW / Bowled bonus | +8 | On top of wicket points |
| 3-wicket bonus | +4 | |
| 4-wicket bonus | +8 | |
| 5-wicket bonus | +12 | |
| Maiden over | +12 | |

**Economy Rate bonus/penalty** (min 2 overs bowled):
| Economy Rate | Points |
|---|---|
| Below 5.00 | +6 |
| 5.00 – 5.99 | +4 |
| 6.00 – 7.00 | +2 |
| 7.01 – 9.99 | 0 |
| 10.00 – 11.00 | −2 |
| 11.01 – 12.00 | −4 |
| Above 12.00 | −6 |

### 6.3 Fielding

| Event | Points | Notes |
|---|---|---|
| Catch | +8 | Awarded to catching fielder |
| 3-catch bonus | +4 | One-time bonus regardless of total catches |
| Stumping | +12 | Wicketkeeper only |
| Run out (direct hit) | +12 | Single fielder touched ball after delivery |
| Run out (not direct hit) | +6 each | Last 2 fielders who touched the ball each get 6 pts |

### 6.4 Other

| Event | Points |
|---|---|
| Announced Starting XI | +4 |
| Concussion / Impact Player sub | +4 |

### 6.5 Captain & Vice Captain Multipliers
- **Captain:** 2× total gameweek points — always.
- **Vice Captain:** 2× total gameweek points **only if the Captain did not play** (absent from the Playing XI and did not appear as an Impact Sub in any match across the entire gameweek). If the Captain plays, the VC earns standard 1× points — no multiplier.
- Applied after base points are calculated and bench subs are resolved.

### 6.6 Multi-Match Accumulation
If a player plays multiple IPL matches in one gameweek, points accumulate across all matches before multipliers are applied.

*Example: Match 1 = 45 pts + Match 2 = 60 pts = 105 pts base, then captain 2× = 210 pts.*

## 7. Strategy Chips

2 chips available per manager per season. Each chip is usable **once per season**. Only one chip can be active per gameweek. Must be selected before lineup lock.

| Chip | Effect | Icon |
|---|---|---|
| **Power Play Bat** | All players with BAT role get their total points doubled (2×) for the gameweek | 🏏 |
| **Bowling Boost** | All players with BOWL role get their total points doubled (2×) for the gameweek | 🎳 |

### Chip + Multiplier Stacking (multiplicative)
| Scenario | Multiplier |
|---|---|
| Captain (BAT) + Power Play Bat active | 2× × 2× = **4×** |
| Captain (BOWL) + Bowling Boost active | 2× × 2× = **4×** |

### Chip Rules
- Chip selection is locked at gameweek lock time
- Used chips are permanently unavailable for the rest of the season
- Confirmation modal required before chip is activated (warns it cannot be undone once GW starts)

## 8. Leaderboard

### Ranking Logic
1. **Primary:** Total season points (descending)
2. **Tiebreaker:** Highest single gameweek score (descending)

### Update Frequency
- Live updates within a gameweek (recalculates after each match is scored)
- Finalised at gameweek end after all bench subs, multipliers, and chip effects are applied

## 9. Data Ingestion

### Source
**SportMonks Cricket API** — Major plan (€29/mo), confirmed IPL 2026 coverage, ball-by-ball data, composable includes.

**Fallback:** Admin can upload match stats manually via CSV if API is unavailable.

### Trigger Mechanism
**Hybrid approach:**
- **Primary:** Admin taps "Import Scores" button in admin panel after each match completes
- **Safety net:** Daily midnight Vercel Cron catches any missed matches

### Key API Fields Required
| Include | Field | Used For |
|---|---|---|
| `batting` | `score`, `ball`, `four_x`, `six_x`, `rate` | Run points, milestones, SR calculation |
| `batting` | `wicket_id` | Duck detection, LBW/bowled bonus |
| `batting` | `catch_stump_player_id`, `runout_by_id` | Fielding point attribution |
| `bowling` | `wickets`, `overs`, `medians`, `runs`, `rate` | Bowling points, ER calculation |
| `lineup` | `position.name`, `lineup.substitution` | Role classification, Starting XI bonus, Impact Player |
| `balls` | Ball-by-ball data | Dot ball calculation |

### Scoring Pipeline Order
1. Import match stats from SportMonks API
2. Calculate base fantasy points per player per match
3. Aggregate player points across all GW matches
4. Apply bench auto-substitutions (end of gameweek)
5. Apply Captain / VC multipliers
6. Apply chip effects
7. Sum team totals
8. Update leaderboard

## 10. Technical Architecture

### Stack
| Layer | Technology |
|---|---|
| Frontend | Next.js 15 + React + TypeScript |
| Styling | Tailwind CSS |
| Backend | Next.js API Routes |
| Database | Neon PostgreSQL (free tier — 0.5GB) |
| ORM | Prisma |
| Auth | Auth.js (OAuth + credentials) |
| Deployment | Vercel Hobby (free) |
| Cron | Vercel Cron (daily midnight) |
| Cricket Data | SportMonks Cricket API |

### Core Services (within Next.js monolith)
1. **Match Import Service** — Polls SportMonks API, stores raw scorecard data
2. **Stat Parser** — Extracts per-player performance from raw API response
3. **Fantasy Points Engine** — Applies scoring rules, calculates base points per player per match
4. **Gameweek Aggregator** — Runs bench subs, applies multipliers and chips, computes team totals
5. **Leaderboard Service** — Rankings, season totals, GW history
6. **Lineup Validation Service** — Enforces squad size, player uniqueness within league, lock timing

### Database Entities (key)
| Entity | Purpose |
|---|---|
| `User` | Auth. Email, name, platform role (USER/ADMIN) |
| `League` | Competition container. Invite code, admin reference, settings |
| `Team` | Manager's team in a league. `totalPoints` incremented at GW end |
| `TeamPlayer` | Squad membership. Stores auction `purchasePrice`. Unique per league |
| `Player` | Real IPL player from API. SportMonks ID, name, team, role |
| `Gameweek` | Weekly period. `lockTime`, `status`, `aggregationStatus` (atomic lock for GW-end) |
| `Match` | IPL fixture. `scoringStatus` tracks pipeline state (scheduled → completed → scoring → scored) |
| `Lineup` | Weekly submission per team per GW |
| `LineupSlot` | Player slot within a lineup. Type (XI/BENCH), bench priority, role (CAPTAIN/VC/TRIPLE_CAPTAIN) |
| `PlayerPerformance` | Per-player per-match stats and computed fantasy points |
| `GameweekScore` | Per-team GW total after all adjustments |

### Infrastructure Cost
**Phase 1 target: £0/month**
- Vercel Hobby: free (frontend, API routes, daily cron)
- Neon PostgreSQL: free tier (0.5GB, 100 compute-hrs/mo)
- SportMonks API: €29/mo (only external cost)

## 12. Phase 1 Scope vs Phase 2+

### Phase 1 (Current)
- Private leagues, invite code join
- Manual auction — admin uploads rosters via CSV
- Weekly lineup management (XI, Captain, VC, bench)
- 2 strategy chips — Power Play Bat and Bowling Boost (once per season each)
- Post-match scoring via SportMonks API
- Bench auto-substitution
- Season leaderboard with GW history
- Admin score import trigger + daily cron safety net

### Phase 2+ (Future)
- Real-time auction engine (WebSockets, $100M budget, $1M starting price, 10-second timer)
- Mid-season auction (after 30 IPL matches)
- Dynamic player market pricing
- Player trading between managers
- Public leagues
- Push/email notifications (lineup lock, GW results)
- Head-to-head matchup mode
- Player injury/availability status surface
- AI lineup suggestions and advanced analytics

## 13. Open Issues

### Game Mechanics
| # | Issue | Detail |
|---|---|---|
| 1 | No role composition constraints | Managers could field 11 batsmen. Consider min 1 WK, 3 BAT, 3 BOWL, 1 ALL. |
| 2 | Impact Sub undefined | Mockup shows Impact Sub slot but rules not specified — when can it be used, does it replace a bench slot? |
| 3 | Chip balance | Power Play Bat benefits more players (5–6 batters) than Bowling Boost (4–5 bowlers) in a typical XI. May be unbalanced — monitor over the season. |
| 4 | No mid-season squad flexibility | Injured players strand managers with no recourse. Consider a simple free-agent swap mechanic. |
| 5 | Captain/VC both absent | No multipliers for anyone. VC only gets 2× if Captain is absent — if both are absent, no multipliers apply. Consider auto-promoting highest scorer as fallback. |

### Missing Features
| # | Feature | Priority |
|---|---|---|
| 6 | Notifications | Lineup lock alerts, GW result notifications |
